Overview

Sandbox Runner is a full-stack platform for securely executing code inside isolated Docker containers. The platform supports Python, JavaScript, Kotlin, Java and Go.

The project focused heavily on infrastructure engineering, container orchestration, runtime isolation and clean backend architecture using Kotlin and Ktor.

Problem Statement

Code execution systems require strong isolation, runtime control and stable resource limitations in order to safely execute untrusted or faulty code.

The goal was to build a modern sandbox architecture with containerized infrastructure and persistent execution storage.

Architecture Approach

Docker Runtime Isolation

Every execution runs inside an isolated Docker container with disabled networking, CPU/memory limits and PID restrictions.

Kotlin + Ktor Backend

The backend is built with Kotlin and Ktor using a clearly separated API, domain, infrastructure and persistence architecture.

Persistent Storage

All executions including stdout, stderr, runtime metrics and status information are stored in PostgreSQL.

Modern Frontend

The React frontend integrates Monaco Editor, runtime statistics, run history and detailed execution views.

Challenges

The biggest challenge was implementing reliable, secure and performant containerized code execution.

Runtime isolation, Docker orchestration, resource limits and language-specific execution flows were particularly complex parts of the system.

Learnings

The project highlighted how important clean infrastructure boundaries, controlled runtime environments and robust architecture are when building executable systems.
